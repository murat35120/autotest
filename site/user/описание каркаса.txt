каркас состоит из
1) страницы index.html в которой ссылки на все дополнительные файлы и title страницы
2) carcass.js & carcass.css - содержит функции и стили для создания элеменов страницы,
обработчик событий, функции связи с сервером и работы с локальным хранением, 
объявлен объект blocks  - мосто хранения ссылок на все созданные элемены страницы.
carcass_functions.saveLS() - сохраняем объект абонент в локальное хранилище
3) заготовка файла script.js с  объявленными но изначально пустыми объектами: 
links - для специфичной реакции на действия пользователя, 
control - для обработки событий пользователя 
	имя метода соответствут dataset.click блока по которому кликнули, 
	dataset.read  содержимое которого изменили
functions - функции как часть кода, 
recivers  - каждой команде API своя функция обработчик,
make_block - создание повторяющихся блоков - используем только наследуемые имена, 
carcass - создане каркаса страницы, 
готовая функция start и ее вызов.
4) для отправки на сервер используем comm.ax_post первый параметр передаваемый объект



пожелания
protocol - !!! это не каркас - это шаблон!!! функции соответствуют командам и ответам протокола связи с сервером. Сюда переносим то что лежит в control и recivers. Содержит связь значений для отправки/получения и мест хранения на клиенте. Можно выделить в отдельный файл 
имена полей в создаваемых блоках - должны наследоваться пудем автоматического добавления префикса. Необходимо для копирования блоков.

мультигруппа
родитель кнопок содержитимена групп через пробел или другой разделитель
item - уникальное имя своей группы,  call - имя группы по клику (набор групп)
связанные элементы группы item могут находиться в любых местах
